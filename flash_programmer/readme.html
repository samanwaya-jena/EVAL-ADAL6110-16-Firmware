1. Overview

The device programmer interface application (DPIA) implements the interface between the CrossCore Embedded Studio Device Programmer utility and the on-board device.

2. Software configuration

  • Board revision : _BF707_EZBOARD_REV_X_Y_ is the format used where X is the major rev and Y is the minor rev. This is defined in the Project Properties dialog which can be accessed by going to the "Project" menu and choosing "Properties", then "C/C++ Build", then "Settings", then "Tool Settings" tab, then "CrossCore Blackfin C/C++ Compiler", then "Preprocessor". There you will find the "Preprocessor definitions". If your exact board rev is not referenced in the source then just use the closest revision.

3. Hardware Requirements and Setup

3.1 Supported platforms

4. Operation Description

WARNING:The CLDP cannot be used if CCES is actively connected to a target board.

  • A pre-built dpia already exists in the same location as this readme. Otherwise, start CCES and using the Example Browser select the W25Q32BV DPIA.
  • Build the project to create the executable that will be used by the command line device programmer(CLDP.exe).
    • NOTE: The cldp.exe utility can be found in the root of the CrossCore Embedded Studio folder.

  • Now the CLDP along with the DPIA can be used to communicate with the flash device.
  • For more information on the CLDP, search for device programmer in CCES Help.
  • Note, there are many options when using the CLDP (including defining your connection type, driver file, loader file, etc. Here is a sample command line to program the a POST loader file to the parallel flash on the EZ-Board using an HPUSB-ICE emulator:

    • cldp -proc ADSP-BF707 -emu 1000 -driver "C:\BF707_EZ-Board\Examples\Device_Programmer\serial\bf707_w25q32bv_dpia.dxe" -cmd prog -erase affected -format hex -file "C:\BF707_EZ-Board\Examples\Power_On_Self_Test\Debug\bf707_post.ldr"
